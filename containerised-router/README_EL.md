## 🎯 Επισκόπηση Έργου

Αυτό το έργο παρουσιάζει μια προσομοίωση δικτύου με κοντέινερ που περιλαμβάνει έναν δρομολογητή που συνδέει δύο απομονωμένα δίκτυα. Έχει σχεδιαστεί για να προσομοιώνει σενάρια τοπικού δικτύου χρησιμοποιώντας κοντέινερ Docker.

### Κύρια χαρακτηριστικά

- **Δρομολογητής Δυο-Δικτύου**: Ένα κοντέινερ δρομολογητή συνδέει δύο ξεχωριστά δίκτυα Docker (network-one και network-two)
- **Προσομοίωση Δικτύου Πελάτη**: Δύο κοντέινερ πελάτη, ένα σε κάθε δίκτυο, προσομοιώνουν την επικοινωνία ακραίων σημείων
- **Επικοινωνία Διασταυρούμενου Δικτύου**: Οι πελάτες μπορούν να επικοινωνούν μεταξύ δικτύων μέσω του δρομολογητή
- **Υποστήριξη NAT**: Η μετάφραση διευθύνσεων δικτύου επιτρέπει στα κοντέινερ να φτάσουν σε εξωτερικά δίκτυα/διαδίκτυο
- **Σύνδεση SSH**: Ασφαλή πρόσβαση κελύφους μεταξύ κοντέινερ για απομακρυσμένη διαχείριση
- **Ολοκληρωμένη Δοκιμή**: Αυτοματοποιημένη σουίτα δοκιμής που επαληθεύει τη σύνδεση και τη διαμόρφωση δικτύου

### Περιπτώσεις Χρήσης

- Μάθηση εννοιών δικτύωσης Docker
- Δοκιμή κανόνων δρομολόγησης δικτύου και firewall
- Ανάπτυξη σεναρίων προσομοίωσης δικτύου
- Κατανόηση iptables και διαμόρφωσης NAT
- Εξάσκηση διαχείρισης κοντέινερ

### Λεπτομέρειες Κοντέινερ

| Στοιχείο | Εικόνα | Δίκτυα | Σκοπός |
|---------|-------|--------|--------|
| **router** | `network-router:latest` | bridge-net, network-one, network-two | Δρομολογεί κίνηση μεταξύ δικτύων, NAT, κανόνες iptables |
| **client1** | `network-client:latest` | network-one | Πελάτης στο πρώτο δίκτυο (172.18.0.3) |
| **client2** | `network-client:latest` | network-two | Πελάτης στο δεύτερο δίκτυο (172.19.0.3) |

### Τύποι Δικτύων

| Δίκτυο | Σκοπός | Υποδίκτυο | Πύλη |
|--------|--------|-----------|------|
| **bridge-net** | Προσομοίωση διαδικτύου/εξωτερικής πρόσβασης | 172.20.0.0/16 | 172.20.0.1 |
| **network-one** | Πρώτο απομονωμένο δίκτυο | 172.18.0.0/16 | 172.18.0.1 |
| **network-two** | Δεύτερο απομονωμένο δίκτυο | 172.19.0.0/16 | 172.19.0.1 |

### Βήμα 2: Δημιουργία Εικόνων Docker

Η διαδικασία δημιουργίας χειρίζεται αυτόματα από το `setup.sh`, αλλά μπορείτε επίσης να δημιουργήσετε χειροκίνητα:

```bash
# Δημιουργία εικόνας δρομολογητή
docker build -f Dockerfile.router -t network-router:latest .

# Δημιουργία εικόνας πελάτη
docker build -f Dockerfile.client -t network-client:latest .
```

### Λεπτομέρειες Εικόνων

#### Εικόνα Δρομολογητή (Dockerfile.router)

**Βάση**: Ubuntu 22.04

**Εγκατεστημένα Πακέτα**:
- `iproute2` - Προχωρημένη δρομολόγηση IP
- `iptables` - Κανόνες firewall και NAT
- `openssh-server` & `openssh-client` - Σύνδεση SSH
- `iputils-ping` - Διαγνωστικά δικτύου
- `net-tools` - Βοηθήματα δικτύου (ifconfig, route)
- `curl` - Πελάτης HTTP
- `vim` - Επεξεργαστής κειμένου

**Διαμόρφωση**:
- Ενεργοποιεί προώθηση IP (`net.ipv4.ip_forward=1`)
- Διαμορφώνει SSH (root login, password auth)
- Ορίζει κωδικό πρόσβασης root σε `router`
- Αντιγράφει και εκτελεί `/entrypoint-router.sh`

#### Εικόνα Πελάτη (Dockerfile.client)

**Βάση**: Ubuntu 22.04

**Εγκατεστημένα Πακέτα**:
- `openssh-server` & `openssh-client` - Σύνδεση SSH
- `iputils-ping` - Διαγνωστικά δικτύου
- `net-tools` - Βοηθήματα δικτύου
- `curl` - Πελάτης HTTP
- `vim` - Επεξεργαστής κειμένου
- `sshpass` - Μη διαδραστική ταυτοποίηση κωδικού SSH

**Διαμόρφωση**:
- Διαμορφώνει SSH (root login, password auth)
- Ορίζει κωδικό πρόσβασης root σε `client`
- Εκτελεί demon SSH ως κύρια διαδικασία

### Εικόνα Windows 11 (Dockerfile.win11)

Αυτή η προαιρετική εικόνα χρησιμοποιείται για την προετοιμασία ενός
περιβάλλοντος μέσων εγκατάστασης Windows 11 μέσα σε κοντέινερ. Ξεκινά από
`scratch` με QEMU, εγκαθιστά βοηθητικά πακέτα και τοποθετεί τα σενάρια build
στον κατάλογο `/run`.

- **Υποστηριζόμενες αρχιτεκτονικές**: `amd64`, `arm64` (αναγνωρίζονται μέσω
  `TARGETARCH` κατά τη διάρκεια του build)
- **Παράδειγμα δημιουργίας**:
  ```bash
  docker build -f dockerfile-win11 -t win11:latest .
  ```

> Σημείωση: μπορείτε να περάσετε `--build-arg VERSION_ARG=<version>` για να
> αλλάξετε την ετικέτα της βασικής εικόνας (π.χ. προσαρμοσμένο
> `dockurr/windows-arm`).

### Εκτέλεση του κοντέινερ Windows 11

```bash
docker run -d \
    --name win11 \
    -p 3389:3389 \   # θύρα RDP
    -p 8006:8006 \   # προαιρετική web κονσόλα
    -v $PWD/win-storage:/storage \  # μόνιμη αποθήκευση
    -e RAM_SIZE=4G \
    -e CPU_CORES=2 \
    -e DISK_SIZE=64G \
    win11:latest
```

Μπορείτε επίσης να ορίσετε τη μεταβλητή `VERSION` (προεπιλογή "11") ή να
δώσετε `VERSION_ARG` κατά το build για να χρησιμοποιήσετε διαφορετική ετικέτα
βάσης.

**Σύνδεση**: αφού τρέξει το κοντέινερ, ανοίξτε έναν πελάτη RDP και συνδεθείτε στο
`localhost:3389`. Το εικονικό Windows που εκκινεί μέσα θα παρουσιάσει την
κανονική οθόνη εγκατάστασης/σύνδεσης. Ο κωδικός ορίζεται κατά την πρώτη
εκκίνηση, όπως σε κανονική εγκατάσταση.

## ⚙️ Ρύθμιση και Διαμόρφωση

### Αυτοματοποιημένη Ρύθμιση (Συνιστάται)

```bash
bash setup.sh
```

Αυτό το σενάριο εκτελεί όλα τα βήματα ρύθμισης αυτόματα:

1. Καθαρίζει τυχόν υπάρχοντα πόρους
2. Δημιουργεί εικόνες Docker
3. Δημιουργεί δίκτυα Docker
4. Ξεκινά κοντέινερ δρομολογητή
5. Συνδέει δρομολογητή σε όλα τα δίκτυα
6. Ξεκινά κοντέινερ πελάτη
7. Διαμορφώνει στατικές διαδρομές στους πελάτες
8. Εμφανίζει περίληψη διαμόρφωσης

### Χειροκίνητη Ρύθμιση (Για Προχωρημένους)

Εάν χρειάζεστε περισσότερο έλεγχο, ακολουθούν τα μεμονωμένα βήματα:

#### 1. Δημιουργία Δικτύων

```bash
docker network create --driver bridge network-one
docker network create --driver bridge network-two
docker network create --driver bridge bridge-net
```

#### 2. Εκκίνηση Κοντέινερ Δρομολογητή

```bash
docker run -d \
  --name router \
  --network bridge-net \
  --cap-add=NET_ADMIN \
  --cap-add=SYS_ADMIN \
  --hostname router \
  --dns 8.8.8.8 \
  --dns 1.1.1.1 \
  network-router:latest
```

Ο δρομολογητής χρειάζεται δυνατότητα `NET_ADMIN` για:
- Φόρτωση κανόνων iptables
- Διαμόρφωση δρομολόγησης
- Ενεργοποίηση προώθησης IP

#### 3. Σύνδεση Δρομολογητή σε Όλα τα Δίκτυα

```bash
docker network connect network-one router
docker network connect network-two router
```

#### 4. Εκκίνηση Κοντέινερ Πελάτη

```bash
docker run -d \
  --name client1 \
  --network network-one \
  --hostname client1 \
  --cap-add=NET_ADMIN \
  --dns 8.8.8.8 \
  --dns 1.1.1.1 \
  network-client:latest

docker run -d \
  --name client2 \
  --network network-two \
  --hostname client2 \
  --cap-add=NET_ADMIN \
  --dns 8.8.8.8 \
  --dns 1.1.1.1 \
  network-client:latest
```

#### 5. Διαμόρφωση Διαδρομών για Επικοινωνία Διασταυρούμενου Δικτύου

```bash
# Στον client1: διαδρομή προς network-two μέσω δρομολογητή
docker exec client1 route add -net 172.19.0.0 netmask 255.255.0.0 gw 172.18.0.2

# Στον client2: διαδρομή προς network-one μέσω δρομολογητή
docker exec client2 route add -net 172.18.0.0 netmask 255.255.0.0 gw 172.19.0.2
```

## 🔧 Αντιμετώπιση προβλημάτων

### Το Κοντέινερ Δεν Θα Ξεκινήσει

```bash
# Ελέγχος αρχείων κατάστασης κοντέινερ
docker logs router
docker logs client1

# Έλεγχος εάν η θύρα χρησιμοποιείται ήδη
docker ps -a

# Αφαίρεση κολλημένων κοντέινερ
docker rm -f router client1 client2

# Επανεκκίνηση με setup.sh reset
bash setup.sh reset
```

### Το Ping Αποτυγχάνει Μεταξύ Δικτύων

**Συμπτώματα**: `ping 172.19.0.3` από client1 αποτυγχάνει

**Πιθανές Αιτίες**:

1. **Διαδρομές δεν διαμορφώθηκαν**
   ```bash
   # Ελέγχος διαδρομών στο client1
   docker exec client1 route -n
   
   # Επανα-διαμόρφωση διαδρομών
   bash setup.sh reset
   ```

2. **Ο δρομολογητής δεν προωθεί**
   ```bash
   # Ελέγχος προώθησης IP στον δρομολογητή
   docker exec router cat /proc/sys/net/ipv4/ip_forward
   # Θα πρέπει να εξάγει: 1
   
   # Ελέγχος κανόνων iptables
   docker exec router iptables -L -v
   # Θα πρέπει να εμφανίζει αλυσίδες FORWARD που δέχονται κίνηση
   ```

3. **Το δίκτυο δεν είναι συνδεδεμένο στον δρομολογητή**
   ```bash
   # Ελέγχος συνδεδεμένων δικτύων
   docker inspect router --format='{{json .NetworkSettings.Networks}}'
   
   # Επανασύνδεση εάν απαιτείται
   docker network connect network-one router
   docker network connect network-two router
   ```

### Η Σύνδεση SSH Αποτυγχάνει

**Συμπτώματα**: `sshpass -p "client" ssh root@172.19.0.3` αποτυγχάνει

**Πιθανές Αιτίες**:

1. **sshpass δεν εγκατάστηκε στην εικόνα πελάτη**
   ```bash
   # Επαλήθευση sshpass στην εικόνα
   docker exec client1 which sshpass
   
   # Προσθήκη στο Dockerfile.client και ξαναδημιουργία
   # Ήδη περιλαμβάνεται στην τρέχουσα έκδοση
   ```

2. **Ο demon SSH δεν εκτελείται**
   ```bash
   # Ελέγχος demon SSH
   docker exec client1 ps aux | grep sshd
   
   # Επανεκκίνηση SSH
   docker exec client1 service ssh restart
   ```

3. **Λάθος κωδικό πρόσβασης**
   - Κωδικό πρόσβασης δρομολογητή: `router`
   - Κωδικό πρόσβασης πελάτη: `client`

### setup.sh

**Σκοπός**: Αυτοματοποιημένη ανάπτυξη και διαμόρφωση δικτύου

- `create_networks()` - Δημιουργεί τρία δίκτυα Docker
- `start_router()` - Ξεκινά κοντέινερ δρομολογητή με δυνατότητες
- `start_clients()` - Ξεκινά δύο κοντέινερ πελάτη

## 🎓 Πόροι Μάθησης

### Βασικές Έννοιες

1. **Δικτύωση Docker**
   - Δίκτυα γέφυρας: Απομονωμένα δίκτυα για επικοινωνία κοντέινερ
   - Οδηγοί δικτύου: Πώς συνδέονται τα κοντέινερ
   - Μοντέλο δικτύωσης κοντέινερ: Πώς εκχωρούνται διευθύνσεις IP

2. **Δρομολόγηση IP**
   - Στατικές διαδρομές: Χειροκίνητα διαμορφωμένες διαδρομές δρομολόγησης
   - Πίνακες διαδρομών: Πώς προωθούνται τα πακέτα
   - Πύλες: Προεπιλεγμένα σημεία δρομολόγησης

3. **Μετάφραση Διευθύνσεων Δικτύου (NAT)**
   - MASQUERADE: Πηγή NAT για εξερχόμενη κίνηση
   - Προώθηση θύρας: Αντιστοίχιση εξωτερικών θυρών σε εσωτερικές υπηρεσίες
   - Παρακολούθηση σύνδεσης: Διατήρηση κατάστασης για διδροϊκή επικοινωνία

4. **iptables**
   - Τύποι πίνακα: filter, nat, mangle, raw
   - Αλυσίδες: INPUT, OUTPUT, FORWARD, PREROUTING, POSTROUTING
   - Κανόνες: Πώς επεξεργάζονται τα πακέτα

5. **SSH**
   - Ταυτοποίηση δημόσιου κλειδιού
   - Ταυτοποίηση κωδικού πρόσβασης
   - Αρχεία διαμόρφωσης SSH
